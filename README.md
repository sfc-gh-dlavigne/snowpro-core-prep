# SnowPro Core COF-C03 Exam Prep

A Streamlit study app for the Snowflake SnowPro Core Certification (COF-C03).  
Questions, answers, and explanations are generated by **Snowflake Cortex** — authored with `claude-opus-4-8` and fact-checked against real Snowflake documentation with `claude-sonnet-4-6` — then cached locally so each question is generated only once.

---

## Features

- **Practice Exam** — Timed, weight-stratified exam scored on a 1000-point scale. Choose 10, 25, 50, 100 (default, matches the real exam), or 150 questions; 100 questions = 115 min
- **Study by Domain** — Work through each domain with instant feedback, explanations, and citations
- **Flashcards** — Key concepts across all 5 domains
- **Progress Tracking** — Accuracy by domain and full exam history; persists across restarts. Every finished exam is reviewable question-by-question (your answer vs. the correct answer, explanation, and citations) with an All/Incorrect/Correct filter
- **Quick Reference** — Cheat sheets for all 5 COF-C03 domains
- **LLM-generated Q&A** — Questions are generated via Cortex COMPLETE (`claude-opus-4-8`) and verified against real Snowflake documentation using `claude-sonnet-4-6`; cached locally so generation only happens once per question
- **Pre-load Cache** — Bulk-generate and cache all questions before a timed exam so it runs without interruption; generates in batches with parallel HTTP and parallel Cortex verification
- **Challenge a question** — Disagree with a graded answer in Study mode? Submit your reasoning and Cortex re-checks it against the question's own cited documentation. If your challenge holds up, the cached question (answer key + explanation) is corrected and your score is updated.

## Requirements

- Python 3.10+
- A Snowflake account with access to **Snowflake Cortex** (`SNOWFLAKE.CORTEX.COMPLETE`)
- The two Cortex models the app uses must be available in your account's region:
  - `claude-opus-4-8` — question generation (`GEN_MODEL`)
  - `claude-sonnet-4-6` — citation verification + challenge adjudication (`VERIFY_MODEL`)

  If these models aren't available in your region, either enable **Cortex cross-region inference**:

  ```sql
  ALTER ACCOUNT SET CORTEX_ENABLED_CROSS_REGION = 'ANY_REGION';
  ```

  or edit the `GEN_MODEL` / `VERIFY_MODEL` constants near the top of `snowpro_study_app.py` to models available to you.
- A role with `USAGE` on a warehouse and `USAGE` on the `SNOWFLAKE` database

## Option A — Run Locally

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure your Snowflake connection

The app resolves a connection in this order (first match wins):

**1. `SNOWFLAKE_CONNECTION_NAME` environment variable (highest priority)**

The name of a connection defined in `~/.snowflake/connections.toml`. Pass it inline so the value reaches the Streamlit process:

```bash
SNOWFLAKE_CONNECTION_NAME=my-connection streamlit run snowpro_study_app.py
```

**2. `.streamlit/secrets.toml` (if present)**

A `[connections.snowflake]` block — an explicit, app-scoped choice, so it is preferred over the default connection when present (and it's the path Streamlit in Snowflake uses automatically). Copy the template and fill it in:

```bash
cp .streamlit/secrets.toml.example .streamlit/secrets.toml
# edit with your account / user / auth / warehouse
```

The connection name is **not** stored in `secrets.toml` — it only selects a section of `connections.toml`.

**3. Default connection (no env var, no secrets.toml)**

If neither of the above is set, the app uses your default connection — `default_connection_name` in `~/.snowflake/config.toml`:

```toml
# ~/.snowflake/config.toml
default_connection_name = "my-connection"
```

The app resolves that name and connects to it by name (not anonymously), so it can apply its own connection options — see the browser-sign-in note below.

> **Gotcha:** `default_connection_name` must live in **`config.toml`**, *not* `connections.toml`. Placing it in `connections.toml` silently breaks default resolution (the connector treats it as a bogus connection) and also breaks `snow connection list`.

Named connections live in `~/.snowflake/connections.toml` and are **case-sensitive** — the value you use above must match a `[section]` exactly:

```toml
# ~/.snowflake/connections.toml
[my-connection]
account   = "xy12345.us-east-1"
user      = "jsmith"
authenticator = "externalbrowser"   # or "oauth_authorization_code", "username_password_mfa", or omit for password
```

> **Note on browser sign-in (OAuth / externalbrowser):** For local runs the app opens the connector connection with the credential cache **disabled** (`client_store_temporary_credential=False`). This means it does **not** read or write the macOS keychain / OS token cache, so you won't hit a keychain-unlock prompt — instead you get a fresh **browser sign-in each time the app starts and on Reconnect**. If your session token expires mid-run, the in-app **Reconnect** button re-runs that browser sign-in without restarting the app. (Password and key-pair auth are unaffected and never prompt a browser.)

### 3. Run

```bash
streamlit run snowpro_study_app.py
```

Open `http://localhost:8501` in your browser.

## Option B — Streamlit in Snowflake (SiS)

1. In Snowsight, go to **Projects → Streamlit → + Streamlit App**
2. Upload `snowpro_study_app.py` as the main file
3. No connection configuration needed — SiS connects automatically
4. Add the following packages in the SiS package picker:
   - `snowflake-snowpark-python` (already available in SiS)
   - `streamlit` (already available in SiS)
5. Click **Run**

Note: `progress.json` and `questions_cache.json` will not persist between SiS sessions (no local filesystem). Questions will regenerate each session, but this has no effect on the exam experience.

## First run notes

- **Recommended:** Use the **Pre-load Cache** page (sidebar) before taking a timed exam. It bulk-generates all 204 questions (or just the 102 exam-variation questions) in batches so the exam runs without any generation pauses.
- If you skip pre-loading, questions are generated live the first time you open each domain in Study mode (expect 15–30 seconds per question while Cortex generates and verifies citations). Subsequent visits use the local cache.
- If generation fails (e.g. no VPN / Cortex unavailable), the app falls back to a small built-in static question bank automatically.
- If the connection drops mid-session, the warning banner shows a **🔁 Reconnect to Snowflake** button so you can re-authenticate without leaving your exam or study session. It reports the underlying reason if it still can't connect.
- **Reset Exam/Study Stats** (Progress page) clears scores — and, for exam stats, the exam history — without touching the question cache.
- **Reset Questions** (Progress page) clears the question cache and forces fresh generation on next access. It does **not** touch your scores or exam history.

## Challenging a question

Study mode fact-checks itself. On any answered question, expand **🚩 Think this is wrong? Challenge it**, explain why you believe the marked answer is incorrect, and submit.

- An adjudicator (Cortex) fetches the question's **cited Snowflake documentation** and rules on your argument based only on that text — the same grounding used to generate and verify questions.
- It is deliberately **conservative**: it overturns the answer key only when the docs clearly support the change, so an unsubstantiated argument won't earn points.
- **If upheld:** the cached question's answer key and explanation are corrected in place — persisted to `questions_cache.json`, so the fix sticks and travels when you share the app — and your score is amended once (a wrong answer that is now correct gets credited).
- **If not upheld:** you get a short, doc-grounded explanation of why the original answer stands.
- Requires the question to have citations and a live Cortex connection; questions with no citations (or unreachable pages) can't be auto-adjudicated.
